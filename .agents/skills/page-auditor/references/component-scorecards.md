# Component Scorecards
# Scoring rubrics for every landing page component type
# Page Auditor reference — read the relevant scorecard during Phase 2

---

## HOW TO USE THESE SCORECARDS

For each component being audited:
1. Find the matching scorecard below
2. Check each criterion — pass or fail
3. Calculate score: (passed criteria / total criteria) × 10, rounded
4. Document specific evidence for each failure
5. Write a specific fix for each failure

---

## NAVBAR SCORECARD

### Criteria (each worth 1 point)
1. ✓ Logo present and clearly visible
2. ✓ Max 5 navigation links (fewer = better)
3. ✓ CTA button visible on right side
4. ✓ CTA button uses contrast color (stands out from nav background)
5. ✓ Navbar height 60-80px (not too tall, not too thin)
6. ✓ Sticky behavior (stays visible on scroll)
7. ✓ No dropdown menus or mega-menus
8. ✓ Mobile: collapses to hamburger
9. ✓ No distracting elements (no social icons, banners, or promotional text)
10. ✓ CTA copy is action-oriented (not just "Login" or "Menu")

### Common Failures
- **Too many links**: every extra link is a distraction. Kill → keep max 4 items
- **No CTA in nav**: the sticky nav CTA is often the highest-converting element on the page
- **CTA blends in**: if the CTA button looks like a nav link → it won't be clicked
- **Not sticky**: users scroll away from the hero and lose the CTA
- **Dropdown menus**: on landing pages, dropdowns increase decision friction → remove

### Score Interpretation
- 9-10: Professional — matches Linear, Vercel, Notion navbars
- 7-8: Good — one or two refinements
- 5-6: Average — CTA or stickiness likely missing
- Below 5: Fundamental issues — likely too many links or no CTA

## HERO SECTION SCORECARD

The most important component. Score this with extra care.

### Criteria (each worth 1 point)
1. ✓ Headline is outcome-focused (result/benefit, not company description)
2. ✓ Headline passes 3-second clarity test (stranger knows what it does instantly)
3. ✓ Headline size 56-80px desktop / 32-48px mobile
4. ✓ Subheadline present (explains how + who + differentiator)
5. ✓ Primary CTA above the fold
6. ✓ Primary CTA copy is action + outcome (not "Submit" or "Click here")
7. ✓ Hero visual present (product/result shown — not stock photo)
8. ✓ Hero visual shows product with real data (not empty state)
9. ✓ Trust signal present (social proof, rating, user count, logo strip)
10. ✓ Secondary CTA present (lower-commitment option)

### Bonus criteria (if present, note as positive)
- Eyebrow label above headline (creates context)
- Risk-removal micro-copy below CTA ("No credit card required")
- Headline uses negative letter-spacing (-1px to -3px) for premium feel
- Visual has depth (perspective tilt, drop shadow, floating elements)

### Common Failures and Fixes

**Failure: Company-name headline**
Current: "Welcome to EduManager"
Fix: "[Users] can [result] without [pain]"
Example: "Manage 10,000 students without spreadsheets"

**Failure: Feature headline instead of outcome**
Current: "AI-powered university management platform"
Fix: Name the outcome, not the technology
Example: "Grade faster, track attendance automatically, run reports in seconds"

**Failure: Generic stock photo as hero visual**
Current: smiling people at computers / handshake photo
Fix: Show the actual product dashboard/app/result
Why: users subconsciously ask "what will I actually use?" — show them

**Failure: Weak CTA copy**
Current: "Get Started" / "Learn More" / "Submit"
Fix: "[Verb] + [outcome] + [risk removal]"
Example: "Start Managing Students Free — No Card Required"

**Failure: No trust signal in hero**
Current: nothing between CTA and next section
Fix: Add one line: "★★★★★ Trusted by 500+ universities" or logo strip
Why: trust must come immediately after the promise — not 3 sections later

## SOCIAL PROOF BAR SCORECARD

### Criteria
1. ✓ Placed immediately after hero (not buried lower)
2. ✓ Contains logos OR stats OR both
3. ✓ If logos: displayed in grayscale (reduces visual noise)
4. ✓ If numbers: specific (not "thousands" — use "12,847")
5. ✓ Numbers are credible and current
6. ✓ Section is visually minimal (doesn't compete with hero)
7. ✓ Copy is short ("Trusted by teams at:" or just the logos)
8. ✓ Not cluttered (max 6 logos or 4 stats)

### Common Failures

**Failure: Colorful logos fighting each other**
Fix: Convert all to grayscale at 60% opacity — consistent, elegant

**Failure: Round numbers that feel fake**
Current: "10,000+ users"
Fix: Use specific numbers: "11,247 active users as of this month"
Why: specificity = credibility. Round numbers = marketing.

**Failure: Missing entirely**
Fix: Add any one of these:
- 3-4 recognizable company logos
- User count with specificity
- Rating ("★★★★★ 4.9/5 from 200+ reviews")
- Single strong metric ("$2.4M in student fees processed")

## PROBLEM SECTION SCORECARD

### Criteria
1. ✓ Section exists (many pages skip this — critical gap)
2. ✓ Describes problems the TARGET USER actually feels
3. ✓ Uses specific, relatable language (not abstract "inefficiency")
4. ✓ Creates emotional resonance ("You know how...?")
5. ✓ Lists 3 specific pain points (not a wall of text)
6. ✓ Uses icon + short title + 1 sentence per problem
7. ✓ Does NOT immediately pitch the product (let the pain land first)
8. ✓ Naturally transitions into the solution section

### Common Failures

**Failure: Missing entirely**
This is the #1 skipped section. Pages that skip it feel like they're selling
to people who already know they need the product — missing all Level 1-2 visitors.
Fix: Add 3 problem cards before the features section.

**Failure: Vague problems**
Current: "Managing your university is complex"
Fix: Name the specific pain
Example: "Entering 500 grades manually every exam season"
Example: "Calling teachers to confirm attendance because nothing syncs"
Example: "Using 4 different tools that don't talk to each other"

**Failure: Jumps straight to solution**
The problem section must let the user feel understood BEFORE showing the solution.
If you describe the problem and immediately say "that's why we built X" in the same
paragraph — let the problem breathe for a full section first.

## FEATURES SECTION SCORECARD

### Criteria
1. ✓ Features translated to benefits (not feature-only descriptions)
2. ✓ Max 6 features per grid (more = overwhelm)
3. ✓ Each card has: icon + short title + 1-2 sentence benefit
4. ✓ Icons consistent style (all outline OR all filled — not mixed)
5. ✓ Section headline outcome-focused ("Everything you need to X" not just "Features")
6. ✓ Copy uses user language (not technical jargon)
7. ✓ Cards equal height and weight (no one card dominates randomly)
8. ✓ At least one feature card mentions a specific measurable result

### Common Failures

**Failure: Pure feature list with no benefits**
Current card: "AI Search · Powered by GPT-4"
Fix: "Find any student record instantly · Search 10,000 profiles in under a second"

**Failure: Too many features (10+ cards)**
Fix: Pick the 6 that solve the biggest pains. Link to full feature page for the rest.
Why: Hick's Law — more options = slower decisions = more exits

**Failure: Generic section heading**
Current: "Features"
Fix: "Everything [audience] needs to [goal]"
Example: "Everything your academic team needs to run without spreadsheets"

**Failure: Mixed icon styles**
Current: some icons filled, some outline, some colored, some gray
Fix: pick ONE icon library (Lucide, Heroicons, Phosphor) and ONE style. Consistent.

## TESTIMONIALS SCORECARD

### Criteria
1. ✓ Real photos (not avatars, initials, or stock)
2. ✓ Full name (not "J.S." or "Anonymous")
3. ✓ Role AND company ("Head of Academics, MIT")
4. ✓ Quote contains a specific result (number, timeframe, or comparison)
5. ✓ Quote sounds human (not "This platform has revolutionized our workflow")
6. ✓ At least 3 testimonials shown
7. ✓ Testimonials from users similar to target audience
8. ✓ Section headline is audience-acknowledging ("Used by teams who...")
9. ✓ No obviously fake or generic testimonials
10. ✓ Layout: card-based with adequate whitespace

### Scoring Bonus
- Before/after structure in quote (+1)
- Video testimonial (+1)
- Verified badge or LinkedIn link (+0.5)

### Common Failures

**Failure: Generic quote with no result**
Current: "This is a fantastic platform! Highly recommend."
Fix: Prompt users for specific results:
"How many hours per week did this save you?"
"What specific problem did this solve?"
Result: "Reduced our grade entry time from 4 hours to 20 minutes per week."

**Failure: No photo**
Current: initials in a colored circle
Fix: Ask customers for a headshot. Even a LinkedIn profile photo.
Why: photos increase testimonial believability by ~60% (Nielsen research)

**Failure: Wrong audience**
If your target is university administrators but testimonials are from students:
Fix: collect testimonials from the decision-maker persona, not end-users

**Failure: Company name only (no role)**
Current: "John Smith, Harvard University"
Fix: "John Smith, Registrar, Harvard University"
Role adds credibility — it answers "is this person like me?"

## STATS / NUMBERS SECTION SCORECARD

### Criteria
1. ✓ Numbers are large and visually dominant (40-64px)
2. ✓ Each stat has a short label (small, uppercase, muted)
3. ✓ Numbers are specific (not round: "12,847" not "10,000+")
4. ✓ Stats are relevant to the VALUE of the product (not vanity metrics)
5. ✓ Max 4 stats (more = visual noise)
6. ✓ Adequate whitespace between stats
7. ✓ Numbers are credible and defensible

### Good Stats to Show
- Users / customers (with specificity)
- Time saved ("8 hours/week average")
- Tasks processed ("4.2M grades processed")
- Uptime / reliability ("99.97% uptime")
- ROI / result ("$2.1M in fees automated")
- Reviews/rating ("★4.9 from 847 reviews")

### Bad Stats (Vanity — Avoid)
- Coffee cups consumed ☕ (joke stats — undermine professionalism)
- Years in business (nobody cares until it's 10+)
- Lines of code (meaningless to buyers)
- Countries "reached" (if you have 1 user per country)

### Common Failures

**Failure: Missing entirely**
Stats sections are high-authority. Large numbers create momentum and legitimacy.
Fix: Add a 3-4 stat strip between features and testimonials.

**Failure: Round numbers**
Current: "10,000+ students"
Fix: "11,247 students managed across 84 universities"
Specific numbers feel like facts. Round numbers feel like marketing.

## PRICING SECTION SCORECARD

### Criteria
1. ✓ Exactly 3 tiers (not 2, not 4+)
2. ✓ Middle tier is visually highlighted (recommended)
3. ✓ Tiers named by audience, not feature level
4. ✓ Monthly/annual toggle with savings % shown
5. ✓ Each tier has: name + price + audience label + feature list + CTA
6. ✓ Feature list max 6 items per tier
7. ✓ "Most Popular" or "Best Value" badge on recommended tier
8. ✓ Risk-removal copy below CTAs ("Cancel anytime" / "No credit card")
9. ✓ Section heading is value-focused ("Simple pricing" not just "Pricing")
10. ✓ Pricing feels scannable (not a wall of text)

### Common Failures

**Failure: 2 tiers**
Psychology: binary choice creates anxiety ("what if I pick wrong?")
Fix: Add a third tier — either a free/starter below or enterprise above

**Failure: 4+ tiers**
Psychology: analysis paralysis. Visitors can't decide → don't decide → leave.
Fix: Consolidate. Ask: what are the real decision points? Usually 3.

**Failure: Feature-named tiers ("Basic / Pro / Premium")**
These mean nothing without reading the feature list.
Fix: name by WHO: "Solo / Team / University" or "Starter / Growth / Scale"

**Failure: No visual differentiation on recommended tier**
Fix: Give the middle tier a different background (brand color or dark), a badge,
and a larger CTA button. It should stand out from across the room.

**Failure: No risk removal**
Fix: Add below each CTA: "No credit card required" or "Cancel anytime"
This single line has been shown to increase trial signups 15-30%

## FAQ SECTION SCORECARD

### Criteria
1. ✓ FAQ section exists (many pages skip — critical gap)
2. ✓ Questions written from user's perspective ("Can I..." not "We offer...")
3. ✓ Addresses the top 5 objections for this product type
4. ✓ Answers are honest and specific (not marketing copy)
5. ✓ Accordion layout (expand/collapse reduces overwhelm)
6. ✓ First question is open by default (shows the interaction)
7. ✓ Max 10 questions (more = intimidating)
8. ✓ Section heading is human ("Questions? We've got answers.")

### Must-Cover Objections (adapt to product)
- Cost: "Is there a free trial?" / "Can I cancel anytime?"
- Security: "Is my data secure?" / "Where is data stored?"
- Complexity: "How long does setup take?" / "Do I need technical skills?"
- Compatibility: "Does it work on mobile?" / "Does it integrate with X?"
- Commitment: "Am I locked into a contract?"

### Common Failures

**Failure: Missing entirely**
FAQ is one of the highest-converting sections because it kills objections
exactly when they form. Without it, hesitant users leave silently.
Fix: Add 6-8 questions addressing the main barriers to signup.

**Failure: Company-voice questions**
Current: "What does EduManager offer?"
Fix: "What features are included in the free plan?"
Write from the user's POV — they're asking, not you answering a prompt.

**Failure: Vague answers**
Current: "Yes, your data is completely secure."
Fix: "Yes. All data is encrypted at rest (AES-256) and in transit (TLS 1.3).
We are SOC 2 Type II certified. Your data is stored in EU/US servers and
never shared with third parties."
Specificity in security answers massively increases trust.

## FINAL CTA SECTION SCORECARD

### Criteria
1. ✓ Section exists at bottom of page (many pages end at FAQ — big miss)
2. ✓ Visually distinct from rest of page (different background/contrast)
3. ✓ Headline is transformation-focused (not just "Get started")
4. ✓ Subtext reinforces risk removal ("Free to start · No card needed")
5. ✓ CTA button is largest on the page
6. ✓ Minimal elements (no distractions — just headline + sub + button)
7. ✓ Creates emotional resonance (not purely transactional)

### Common Failures

**Failure: Missing entirely**
Users who scroll to the bottom are your most interested visitors.
Not having a final CTA wastes your most qualified leads.
Fix: Add a full-width contrasting section at the bottom.
Even a simple: headline + one CTA button is 10x better than nothing.

**Failure: Repeats hero CTA exactly**
Current: "Start Free Trial" (same as hero)
Fix: The final CTA should speak to someone who just read the whole page.
Use a transformation frame: "Stop managing grades in spreadsheets."
Or a momentum frame: "Join 2,000+ universities who already made the switch."

## LOGIN / SIGNUP FORM SCORECARD

### Criteria
1. ✓ Minimum fields (email only for first step if possible)
2. ✓ Labels above inputs (not placeholder text as labels)
3. ✓ Input height 44-48px (thumb-friendly)
4. ✓ Primary CTA button full-width (on mobile especially)
5. ✓ Error states visible and human (not "Invalid input")
6. ✓ Social login options (Google/GitHub) reduce friction significantly
7. ✓ Password field has show/hide toggle
8. ✓ Success state defined (confirmation, redirect, or message)
9. ✓ Trust signals near form ("SSL secured" / "We never spam")
10. ✓ No CAPTCHA on initial signup (add only if spam is proven problem)

### Field Count Rules
- Email only: ~80% completion rate
- Email + Password: ~65% completion rate
- Email + Password + Name: ~50% completion rate
- Email + Password + Name + Company + Role: ~25% completion rate

Every unnecessary field costs you ~11% of signups.

### Common Failures

**Failure: Placeholder text used as labels**
Current: input placeholder "Enter your email"
Fix: add a label above: "Email address"
Why: when user starts typing, placeholder disappears → they forget what the field is

**Failure: Too many fields for cold signup**
Fix: collect only email on step 1. Get more info during onboarding after they're in.

**Failure: Generic error messages**
Current: "Error: Invalid email"
Fix: "That email doesn't look right — try user@company.com format"

**Failure: No social login**
Fix: Add "Continue with Google" as the primary option
Results: 40-60% of users prefer social login → more completions

## BUTTON / CTA COMPONENT SCORECARD

### Criteria
1. ✓ Copy is action + outcome (not just an action verb alone)
2. ✓ Color contrasts with background (passes 3:1 contrast ratio)
3. ✓ Size: min 44px height (48px preferred)
4. ✓ Adequate padding: min 16px vertical, 24px+ horizontal
5. ✓ Hover state defined (color shift or slight scale)
6. ✓ Active/pressed state defined (slight scale down 0.97)
7. ✓ Loading state defined (spinner + "Loading..." for async actions)
8. ✓ Border radius consistent with rest of design system
9. ✓ No more than one primary button per visual section
10. ✓ Secondary button clearly visually subordinate to primary

### Common Failures

**Failure: Low contrast**
Current: light gray button on white background
Fix: Use brand primary color for CTA. Check contrast at contrast.tools.

**Failure: Too small**
Current: 32px height
Fix: minimum 44px for accessibility and mobile usability

**Failure: No hover/active state**
Fix: hover → brightness(1.1) or translateY(-1px)
active → scale(0.97)
These micro-interactions confirm the button is clickable and the action registered.

## CARD COMPONENT SCORECARD

### Criteria
1. ✓ Consistent padding (24px or 32px — never less than 16px)
2. ✓ Clear visual hierarchy within card (icon → title → description order)
3. ✓ Equal height across grid (no orphan tall cards)
4. ✓ Hover state defined (lift, border color change, or shadow)
5. ✓ Border radius consistent with system (0px / 8px / 12px — pick one)
6. ✓ Content doesn't overflow or truncate awkwardly
7. ✓ CTA within card (if applicable) is visually clear
8. ✓ Cards don't compete with each other (equal visual weight)

### Common Failures

**Failure: Unequal padding**
Current: some cards 12px padding, some 32px
Fix: define one padding value for all cards of the same type. Use CSS variables.

**Failure: No hover state**
Fix: transform: translateY(-4px) + box-shadow increase, 0.25s ease-out
This single interaction makes a page feel significantly more professional.

## MODAL / OVERLAY SCORECARD

### Criteria
1. ✓ Can be dismissed easily (X button, ESC key, backdrop click)
2. ✓ Backdrop darkens page to create focus
3. ✓ Centered vertically and horizontally on desktop
4. ✓ Full-screen on mobile (not a tiny floating box)
5. ✓ Does NOT trigger on page arrival for cold traffic
6. ✓ Clear single purpose (not multiple actions in one modal)
7. ✓ Primary action button is obvious
8. ✓ Destructive actions (delete, cancel) require confirmation

### Common Failures

**Failure: Popup appears immediately on page load**
This is the #1 trust-destroying UX pattern for cold traffic.
Fix: delay minimum 30 seconds OR trigger only on exit intent OR remove entirely.

**Failure: Hard to close**
Fix: X button top-right, ESC key, clicking backdrop — all three methods must work.

## FOOTER SCORECARD

### Criteria
1. ✓ Logo present (reinforces brand)
2. ✓ Organized into columns (Product / Resources / Company / Legal)
3. ✓ Privacy Policy link present
4. ✓ Terms of Service link present
5. ✓ Contact method present
6. ✓ Copyright line at bottom
7. ✓ Text color is muted (footer is not primary content)
8. ✓ No CTAs competing with main page CTA
9. ✓ Social links present (optional but professional)
10. ✓ Consistent with brand visual system

### Common Failures

**Failure: No legal links**
Missing Privacy Policy / Terms kills trust with privacy-conscious users.
In many jurisdictions, Privacy Policy is legally required.
Fix: add both links to footer bottom row. Use a basic generator if needed.

**Failure: Empty or minimal footer**
A nearly empty footer reads as "unfinished startup."
Fix: at minimum — logo + 3-4 link columns + legal row. 30 minutes of work, big trust signal.
