# Psychology Checks
# Full psychology audit checklist for landing pages and components
# Page Auditor reference — read during Phase 4

---

## THE 5 CORE PSYCHOLOGY CHECKS

Run all 5 on every audit, regardless of scope.

---

## CHECK 1: CLARITY AUDIT

### The 3-Second Test
Cover everything except the headline and subheadline.
Ask: can a stranger explain what this product does in 3 seconds?

**Pass criteria:**
- Product category is clear (what type of thing is this?)
- Target user is clear (who is this for?)
- Core outcome is clear (what does the user get?)

**Fail signals:**
- Uses buzzwords: "revolutionary", "game-changing", "next-generation", "innovative"
- Uses jargon the target user wouldn't use themselves
- Describes the company instead of the user's outcome
- Could describe 10 different products (too generic)
- Requires prior knowledge to understand

**Scoring:**
- All 3 pass → Clarity: HIGH
- 2 of 3 pass → Clarity: MEDIUM (identify which fails)
- 1 or 0 pass → Clarity: LOW (critical fix needed)

### The Skim Layer Test
Read ONLY: headlines, bold text, button labels, numbers, image captions.
The skim layer alone must tell the complete story.

**Pass criteria:**
- Page purpose is clear from skim layer alone
- Main benefit communicated without body copy
- CTA is clear without reading surrounding text

**Fail signals:**
- Skim layer reads as disconnected fragments
- Key benefit only in body copy (no user will read)
- CTA button label is generic ("Submit", "Go", "Click here")

## CHECK 2: TRUST AUDIT

### The Trust Stack Check
Is trust built in the right order? Check placement:

| Trust Type | Should Appear At |
|-----------|-----------------|
| Logo/brand quality | Immediate (navbar) |
| Social proof numbers | Within hero or just below |
| Company logos | Just below hero |
| Testimonials | After features |
| Specific stats | After features or alongside |
| Press mentions | After hero if strong |
| Security badges | Near forms and pricing |
| Team/about | About page, or bottom of landing |

**Fail signals:**
- No social proof in first viewport
- Testimonials placed before features (trust before understanding = cart before horse)
- Security badges hidden in footer only (should be near the conversion point)
- Generic testimonials with no names or roles
- No trust signals at all (critical fail)

### The Credibility Check
Does the page feel like a real, established product?

**Pass criteria:**
- Professional visual design (consistent spacing, typography, colors)
- Real photography (not obvious stock photos)
- Specific numbers (not round/vague)
- Real company information available (about, contact, legal)
- No broken elements (images, links, layouts)

**Fail signals:**
- Obvious stock photos (smiling people in offices, handshakes)
- All numbers are round ("10,000+ users" feels made up)
- No legal pages linked (Privacy Policy, Terms)
- Broken images or placeholder text still in page
- Inconsistent design (multiple font sizes, random spacing)

## CHECK 3: FRICTION AUDIT

### The Conversion Path Check
Count every step between "visitor arrives" and "primary action completed."

**Low friction (good):** arrive → read → click CTA → enter email → done (3 steps)
**High friction (bad):** arrive → read → click CTA → fill 7-field form → verify email → onboarding quiz → set up profile → done (7+ steps)

For cold traffic: target 2-3 steps max to first value moment.

### The Form Friction Check
Count form fields. Benchmark:
- 1 field (email only): ~80% completion
- 2 fields: ~70%
- 3 fields: ~55%
- 4+ fields: below 40%

**Pass criteria:**
- First-touch signup asks for email only (or email + password max)
- No fields that could be collected later (name, company, role, phone)
- Social login option available
- No CAPTCHA on initial signup

### The Exit Path Check
How many ways can a user leave the page WITHOUT converting?

Count: navigation links + external links + social media icons + banner links

**Pass criteria for dedicated landing pages:** 0-2 exit paths (logo → home, maybe one nav)
**Pass criteria for homepage/marketing site:** 4-6 focused exit paths max

**Fail signals:**
- Full navigation with 8+ items on a conversion-focused landing page
- Social media icons in header (users click → go to social → never return)
- Unrelated blog or resource links competing for attention

### The Cognitive Load Check
Does the page feel like work?

**Fail signals:**
- Paragraphs longer than 4-5 lines without a break
- More than 3 concepts introduced in the same section
- No visual breathing room between sections
- Dense tables or feature matrices without scannable hierarchy
- Wall of text where bullets/cards would communicate faster

## CHECK 4: AWARENESS LEVEL MATCH

### Detect Traffic Awareness Level
Based on what's known about traffic source:
- Cold social media / display ads → Level 1-2 (Unaware / Problem Aware)
- SEO traffic, branded search → Level 3-4 (Solution / Product Aware)
- Email campaign, retargeting → Level 4-5 (Product Aware / Most Aware)
- Direct / referral → Level 3-4

### Check Hero Copy Against Awareness Level

**Level 1-2 traffic → hero must lead with problem:**
Correct: "Still managing 10,000 students in spreadsheets?"
Incorrect: "EduManager Pro — Start your free trial today"

**Level 3 traffic → hero must lead with differentiation:**
Correct: "The only university platform with built-in automated grading"
Incorrect: "Welcome to EduManager" (too vague)

**Level 4-5 traffic → hero can lead with the offer:**
Correct: "Start Free — Full access for 14 days, no card needed"
Incorrect: Long educational copy (they already know — get out of the way)

### Mismatch Diagnosis
If the page is written for Level 5 but traffic is Level 2:
- Hero leads with offer (visitors don't understand the value yet)
- No problem section (visitors don't feel understood)
- Features section uses jargon (visitors don't have context)
- Result: high bounce rate, low scroll depth, low conversion

Fix: add a problem section, rewrite hero to lead with the pain, then the solution.

## CHECK 5: CONGRUENCY AUDIT

### End-to-End Message Check
The full user journey must feel like ONE conversation:

```
[Traffic source] → [Landing page] → [CTA] → [Thank you page / email]
```

**Check each transition:**

**Traffic source → Hero:**
Does the hero headline use the same language/promise as what sent people here?
- Ad: "Automate university grading" → Hero: "Automated grading for universities" ✓
- Ad: "Automate university grading" → Hero: "AI-powered academic platform" ✗

**Hero → CTA:**
Does the CTA fulfill what the headline promised?
- Headline: "Grade faster" → CTA: "Start grading faster free" ✓
- Headline: "Grade faster" → CTA: "Book a Demo" ✗ (disconnect — demo ≠ grading faster now)

**CTA → Thank you page:**
Does what happens after clicking match the expectation set by the CTA?
- CTA: "Start Free Trial" → Thank you: "Your trial is active!" ✓
- CTA: "Start Free Trial" → Thank you: "Thanks for your interest, we'll be in touch" ✗

### Voice Consistency Check
Read a sentence from the hero and a sentence from the FAQ.
Do they sound like the same person wrote them?

**Pass:** consistent tone throughout (casual stays casual, professional stays professional)
**Fail:** hero is friendly and conversational, FAQ is stiff corporate-speak

## ADDITIONAL PSYCHOLOGY PATTERNS TO CHECK

### Loss Aversion (is it used correctly?)
Loss aversion copy: "Stop losing 8 hours/week to manual grading"
This is valid and powerful when used at:
- Hero headline (sets the stakes)
- Final CTA section (consequence of NOT acting)

Overuse warning: if more than 2-3 sections use fear/loss framing, the page
feels anxiety-inducing rather than empowering. Balance with positive outcomes.

### Anchoring (pricing pages)
Is the highest price shown first? (makes lower tiers feel cheaper)
Is the "old price" shown alongside the "new price"? (makes savings feel real)
Is the value received described before the price? ($500/month in time saved → $29/month)

### Specificity Principle
Count: how many claims on the page use specific numbers vs. vague language?

**Good ratio:** 60%+ of claims are specific
**Bad ratio:** mostly vague ("saves time", "thousands of users", "powerful features")

Specificity scanner — flag these vague phrases for replacement:
- "thousands" / "millions" / "many" → replace with exact number
- "saves time" → replace with "saves X hours/week"
- "easy to use" → replace with "set up in under 15 minutes"
- "powerful" → replace with what specifically it does that's powerful
- "trusted" → replace with who trusts it and how many

### The Premium Perception Check
Before reading a single word, does the page LOOK like a professional product?
This is a gestalt check — overall visual impression in 1 second.

**Premium signals:**
- Consistent spacing (8px grid)
- Max 2 font families
- Max 3 colors in active use
- High-quality visuals (product screenshots with real data)
- Generous whitespace between sections

**Non-premium signals:**
- Cramped layout (sections too close together)
- Multiple font sizes that don't follow a clear hierarchy
- More than 4-5 colors fighting each other
- Obvious stock photos
- Inconsistent border-radius (some elements rounded, some sharp, no system)
- Too many animations (every element animating = carnival feel)
